<objective>
Fix report generation to include checkpoint-resumed scoring data so reports reflect ALL scored opportunities.

Purpose: When the pipeline resumes from a checkpoint, `allScoredResults` only contains the current session's scores. The archived `.pipeline/checkpoint-*.json` files contain the full `ScoringResult` objects from prior sessions, but these are never loaded back. This causes `writeEvaluation` and `writeFinalReports` to produce incomplete reports, forcing users to manually run `regen-reports.ts`. After this fix, resumed runs produce complete reports automatically.

Output: `loadArchivedScores` helper + pipeline-runner integration + tests proving correctness on resume.
</objective>

2. If directory doesn't exist, return `[]`

6. Return `[...map.values()]`

The archive files contain full `ScoringResult` objects keyed by l3Name. The resume file only contains metadata entries (l3Name, status, completedAt). This function reads the ARCHIVE files, not the resume file.

Do NOT validate ScoringResult with Zod -- these are written by our own code and trusted. Simple JSON.parse is sufficient, matching the pattern in regen-reports.ts.
