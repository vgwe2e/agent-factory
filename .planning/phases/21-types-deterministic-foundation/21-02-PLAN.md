**filter.ts** -- Single exported function:

`filterTopN(scored: PreScoreResult[], topN: number): FilterResult`

Algorithm per user decisions:
1. Separate eliminated candidates (eliminationReason !== null) from rankable candidates.
2. Sort rankable candidates by: composite DESC, then aggregatedMaxValue DESC, then l4Id ascending (deterministic).
3. Take initial top-N slice.
4. Find the boundary score = composite of the Nth candidate.
5. Expand: include ALL candidates tied at the boundary score (same composite after 4-decimal rounding).
6. Cap: if expanded set exceeds topN * 1.1 (10% overflow), trim back by breaking ties using aggregatedMaxValue DESC then l4Id ascending. Keep exactly floor(topN * 1.1) survivors.
7. Mark survived=true on survivors, survived=false on the rest.
8. Compute FilterStats: totalCandidates (including eliminated), requestedTopN, actualSurvivors, eliminated count, cutoffScore (boundary composite), tiesAtBoundary (how many candidates share the boundary score).

Edge cases:
- If rankable candidates <= topN, all survive (no filtering needed). cutoffScore = lowest composite.
- If topN <= 0, return all eliminated, no survivors.
- Eliminated candidates always appear in FilterResult.eliminated with survived=false.

**Tests** -- Cover:
- Basic filtering: 10 candidates, topN=3, correct top-3 survive
- Tie at boundary: 2 candidates tied at position 3, both survive (within 1.1x cap)
- Tie overflow cap: many candidates tied, verify cap at floor(topN * 1.1)
- Max_value tiebreaking: candidates with same composite ranked by aggregatedMaxValue DESC
- L4 ID final tiebreaker: deterministic ordering when composite and maxValue match
- Eliminated candidates excluded from ranking but included in result
- Edge: fewer candidates than topN -> all survive
- FilterStats correctness for each scenario

**pre-scorer.ts** -- Single exported function:

`preScoreAll(hierarchy: L4Activity[], topN: number): FilterResult`

Orchestration flow:
1. For each L4Activity in hierarchy:
   a. Compute all 6 dimension scores (call each dimension scorer)
   b. Compute weighted composite via computeDeterministicComposite
   c. Detect red flags via detectL4RedFlags
   d. Apply red flag penalties via applyRedFlagPenalties (may eliminate or penalize composite)
   e. Aggregate max_value: sum of skill.max_value across all skills in the L4
   f. Build PreScoreResult object with l4Id=l4.id, l4Name=l4.name, l3Name=l4.l3, l2Name=l4.l2, l1Name=l4.l1, skillCount=l4.skills.length
   g. Initially set survived=false (filter will update)
2. Pass all PreScoreResult[] to filterTopN(scored, topN)
3. Return the FilterResult

Performance note: This is a single-pass map over L4Activity[] followed by a sort. For 826 items this should complete in <5ms. No async, no I/O.

- Aggregated max_value: L4 with 3 skills -> sum of their max_values
- Empty skills: L4 with no skills -> skillCount=0, appropriate dimension scores
