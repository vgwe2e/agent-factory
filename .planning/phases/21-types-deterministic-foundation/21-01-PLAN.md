**red-flags.ts** -- Two exported functions:

1. `detectL4RedFlags(l4: L4Activity): RedFlag[]` -- Detect 3 red flags per user decision:
   - DEAD_ZONE: decision_exists=false AND every skill has 0 actions + 0 constraints (or skills is empty)
   - NO_STAKES: financial_rating="LOW" AND impact_order="SECOND"
   - CONFIDENCE_GAP: rating_confidence="LOW"
   Use the existing RedFlag tagged union from types/triage.ts. For DEAD_ZONE use `{ type: "DEAD_ZONE", decisionDensity: 0 }`. For NO_STAKES use `{ type: "NO_STAKES", highFinancialCount: 0, allSecondOrder: true }`. For CONFIDENCE_GAP use `{ type: "CONFIDENCE_GAP", lowConfidencePct: 1.0 }`.

2. `applyRedFlagPenalties(composite: number, flags: RedFlag[]): { composite: number; eliminated: boolean; eliminationReason: string | null }` -- Apply per user decision:
   - If flags include DEAD_ZONE: eliminated=true, reason="DEAD_ZONE", composite=0
   - If flags include NO_STAKES: eliminated=true, reason="NO_STAKES", composite=0
   - If flags include CONFIDENCE_GAP (and not already eliminated): composite * 0.3, eliminated=false
   - Hard elimination takes priority over penalty (if both DEAD_ZONE and CONFIDENCE_GAP, eliminate)
   - Round resulting composite to 4 decimal places

PHANTOM and ORPHAN are NOT detected -- per user decision these are L3 properties irrelevant at L4 level.

**Tests** -- Cover:
- DEAD_ZONE detection: no decision + all skills empty actions/constraints
- DEAD_ZONE not triggered: decision_exists=true even with empty skills
- NO_STAKES detection: LOW financial + SECOND impact
- CONFIDENCE_GAP detection: LOW rating_confidence
- Multiple flags on same L4
- applyRedFlagPenalties: elimination returns composite=0, CONFIDENCE_GAP returns composite*0.3, both together eliminates
  </action>
