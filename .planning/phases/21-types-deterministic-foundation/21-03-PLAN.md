**format-pre-score-tsv.ts** -- Single exported pure function following the format-scores-tsv.ts pattern:

`formatPreScoreTsv(results: PreScoreResult[]): string`

1. Sort results by composite DESC (highest first). Eliminated candidates sort to the bottom (composite=0).
2. Header row (tab-separated):
   `rank\tl4_id\tl4_name\tl3_name\tl2_name\tl1_name\tfinancial_signal\tai_suitability\tdecision_density\timpact_order\trating_confidence\tarchetype_completeness\tcomposite\tsurvived\telimination_reason\tred_flags\tskill_count\taggregated_max_value`

3. Each data row:
   - rank: 1-based position after sorting
   - l4_id through l1_name: string fields via tsvCell
   - 6 dimension scores: formatted to 4 decimal places
   - composite: formatted to 4 decimal places
   - survived: boolean via tsvCell (Y/N)
   - elimination_reason: string or empty via tsvCell
   - red_flags: comma-separated flag types (e.g., "DEAD_ZONE,CONFIDENCE_GAP") or empty
   - skill_count: integer
   - aggregated_max_value: number

4. End with trailing newline.

Import tsvCell, tsvRow from "./tsv-utils.js" and PreScoreResult from "../types/scoring.js".

Per user decision: eliminated L4s still appear in the TSV with eliminated status and reason for audit visibility. They sort to the bottom since composite=0.

**Tests** -- Cover:
- Single survivor: correct header + 1 data row
- Mix of survived and eliminated: eliminated at bottom, correct Y/N values
- Elimination reason appears in column
- Red flags formatted as comma-separated types
- Dimension scores rounded to 4 decimal places
- Empty results: just header + newline
- Rank numbers sequential (1, 2, 3...)
