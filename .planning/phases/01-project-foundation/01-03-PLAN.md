    - Test: When Ollama is running but target model not in model list, status includes missing model names
    - Test: checkOllama never calls any non-localhost URL (no cloud dependency)

  <action>
Create `src/infra/ollama.ts`:

Define types:
```typescript
export interface OllamaModel { name: string; size: number; modified_at: string; }
export interface OllamaStatus {
  connected: boolean;
  models: OllamaModel[];
  missingModels: string[];
  error?: string;
}
```

Export `checkOllama(requiredModels?: string[]): Promise<OllamaStatus>`:
- Default requiredModels to ["qwen2.5:7b", "qwen2.5:32b"] (the two-model strategy)
- fetch GET `http://localhost:11434/api/tags` with a 5-second timeout (AbortSignal.timeout(5000))
- On connection error: return { connected: false, models: [], missingModels: requiredModels, error: "Ollama is not running. Start it with: ollama serve" }
- On success: parse response JSON (Ollama returns { models: [{ name, size, modified_at, ... }] })
- Compare available model names against requiredModels. Model names in Ollama include tag (e.g., "qwen2.5:7b") -- match by checking if the available model name starts with the required model name (to handle "qwen2.5:7b-instruct-q4_K_M" matching "qwen2.5:7b")
- Return { connected: true, models: available, missingModels: [...any not found] }

Export `formatOllamaStatus(status: OllamaStatus): string`:
- If not connected: red "Ollama: NOT CONNECTED -- {error}"
- If connected with no missing: green "Ollama: Connected ({N} models available)" + list each model with size in GB
- If connected with missing: yellow "Ollama: Connected but missing models: {list}" + suggest `ollama pull {model}`

Create `src/infra/ollama.test.ts` using node:test:
- Mock fetch globally using node:test mock capabilities to intercept localhost:11434 calls
- Test all behaviors above
- Verify no non-localhost URLs are ever fetched (assert mock was only called with localhost URLs)

Write tests RED, then implement GREEN.
