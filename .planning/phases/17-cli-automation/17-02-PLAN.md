  <action>
    Modify src/cli.ts to implement the full lifecycle:

    **1. Add Commander options (after line 65, the --sim-timeout option):**
    - `.option("--retry <n>", "Retry errored opportunities up to N times at concurrency 1", "0")` — default "0" means no retries
    - `.option("--teardown", "Tear down cloud resources after pipeline completes")` — boolean flag, default false

    **2. Add `retry` and `teardown` to the opts type** in the action handler signature (line 66).

    **3. Validate --retry** (after simTimeout validation, ~line 135):
    - Parse as integer, reject negative values or NaN with error message and `process.exit(1)`

    **4. Restructure the try/finally block (lines 205-233) into a try/catch/finally with retry loop:**

    ```
    try {
      costTracker?.start();

      // Initial pipeline run
      let lastResult = await runPipeline(opts.input, pipelineOptions, logger);

      // Retry loop (AUTO-01)
      const maxRetries = parseInt(opts.retry ?? "0", 10);
      for (let attempt = 1; attempt <= maxRetries; attempt++) {
        if (lastResult.errorCount === 0) break;

        const cleared = clearCheckpointErrors(opts.outputDir);
        console.log(`\n=== Retry ${attempt}/${maxRetries}: ${cleared} errored opportunities ===\n`);

        lastResult = await runPipeline(opts.input, {
          ...pipelineOptions,
          concurrency: 1,  // Always concurrency 1 for retries (locked decision)
        }, logger);
      }

      pipelineResult = lastResult;
    } catch (err) {
      // Fatal failure — ALWAYS tear down to prevent orphaned pods (AUTO-02)
      await doCleanup();
      const msg = err instanceof Error ? err.message : String(err);
      console.error(`${RED}Fatal: ${msg}${RESET}`);
      process.exit(2);  // EXIT CODE 2 = fatal (AUTO-04)
    } finally {
      costTracker?.stop();
      // Teardown only if --teardown flag set (AUTO-02)
      if (opts.teardown) {
        await doCleanup();
      }
    }
    ```

    **5. Import clearCheckpointErrors** at the top of cli.ts:
    `import { clearCheckpointErrors } from "./infra/checkpoint.js";`

    **6. Change teardown behavior (AUTO-02):**
    - Remove the unconditional `await doCleanup()` from the finally block (current line 232)
    - In the finally block: only call `doCleanup()` if `opts.teardown` is set
    - In the catch block (fatal): ALWAYS call `doCleanup()` regardless of flag (don't leave orphaned pods)
    - Signal handlers (SIGINT/SIGTERM, line 178-179) keep their existing behavior (always clean up on signal)

    **7. Replace exit logic at end of action handler (AUTO-04):**
    Replace the current ad-hoc exit (lines 260-266) with:
    ```
    if (pipelineResult.errorCount > 0) {
      console.error(`${RED}${pipelineResult.errorCount} opportunities still errored after ${maxRetries > 0 ? maxRetries + " retries" : "scoring"}.${RESET}`);
      process.exit(1);  // EXIT CODE 1 = errors remain
    }

    console.log(`${GREEN}Results written to ${opts.outputDir}${RESET}`);
    // EXIT CODE 0 = implicit (all scored successfully)
    ```

    **8. Move costTracker.start() BEFORE the retry loop and costTracker.stop() AFTER** so total GPU time spans all retry attempts (pitfall 3 from research).

    **Key constraints per locked decisions:**
    - Retry loop in cli.ts, NOT in pipeline-runner.ts
    - Concurrency forced to 1 for all retry attempts
    - Banner format exactly: `=== Retry N/M: X errored opportunities ===`
    - Re-run full pipeline for retries (checkpoint handles skip logic)
    - Backend created once, shared across all retry attempts

    **Print the Pipeline banner info including retry config:**
    After the existing pipeline info block (~line 201), add:
    ```

      console.log(`Retries:     ${maxRetries} (concurrency 1)`);
    }
    if (opts.teardown) {
      console.log(`Teardown:    enabled`);
    }
    ```
