    1. Import Semaphore from ../infra/semaphore.js

    3. Import createCheckpointWriter from ../infra/checkpoint.js

    5. Create semaphore: `const semaphore = new Semaphore(options.concurrency ?? 1)`
    6. Create checkpoint writer: `const writer = createCheckpointWriter(options.outputDir, checkpoint)`
    7. Create progress tracker: `const progress = createProgressTracker(processable.length, logger)`
    8. Set up periodic progress reporting: `const progressInterval = setInterval(() => progress.report(), 5000)` -- clear in finally block.

    9. Replace the `for` loop with:
    ```typescript
    const tasks = processable.map((triage) => semaphore.run(async () => {
      // Same body as current loop but:
      // - Uses writer.enqueue() instead of checkpoint.entries.push + saveCheckpoint
      // - Wraps callWithResilience in withTimeout((_signal) => callWithResilience(...), requestTimeoutMs)
      // - Calls progress.start/complete/error
      // - Catches TimeoutError specifically and logs it as a timeout
      // - context-tracker addResult/addError calls remain (single-threaded JS event loop means no race on in-memory state)
    }));
    await Promise.allSettled(tasks);
    ```

    10. After allSettled: `clearInterval(progressInterval)`, `writer.flush()`, `progress.report()` (final).

    11. archiveAndReset threshold logic: Remove the sinceLastArchive counter. Instead, call archiveAndReset after allSettled completes (single flush). The context-tracker addResult is safe because JS is single-threaded -- each await point yields but mutations are atomic within a microtask.

    **Important: context-tracker thread safety note.** addResult/addError mutate EvaluationContext (Set, Map, Array). In Node.js single-threaded event loop, these mutations are safe even with concurrent promises because each mutation runs to completion within its microtask. No mutex needed. Document this as a code comment.
