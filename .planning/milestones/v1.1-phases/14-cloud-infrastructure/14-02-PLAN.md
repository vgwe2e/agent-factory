    **Backend factory changes (backend-factory.ts):**

    Change createBackend to be async: `export async function createBackend(backend, options?): Promise<BackendConfig>`

    Extend BackendConfig:
    ```typescript
    export interface BackendConfig {
      chatFn: ChatFn;
      backend: Backend;
      cleanup?: () => Promise<void>;  // teardown for cloud resources
      costTracker?: CostTracker;      // GPU cost tracking (cloud only)
    }
    ```

    Extend VllmBackendOptions to make vllmUrl optional:
    ```typescript
    export interface VllmBackendOptions {
      vllmUrl?: string;        // If omitted and RUNPOD_API_KEY set, auto-provision
      vllmModel?: string;
      runpodApiKey?: string;   // From RUNPOD_API_KEY env var
    }
    ```

    Modify the vllm case in createBackend:
    1. If options.vllmUrl is provided: use it directly (existing behavior -- user manages their own vLLM instance)
    2. If options.vllmUrl is NOT provided AND options.runpodApiKey is set:
       a. Create cloud provider: `const provider = createCloudProvider({ apiKey: options.runpodApiKey })`
       b. Provision endpoint: `const endpoint = await provider.provision()`
       c. Log endpoint ID prominently: `console.log("RunPod endpoint: " + endpoint.endpointId)` (for manual cleanup if needed)
       d. Create cost tracker: `const costTracker = createCostTracker()`
       e. Create cleanup function: `const cleanup = async () => { try { await provider.teardown(endpoint); } catch {} }`
       f. Use endpoint.baseUrl for vLLM client: `createVllmChatFn(endpoint.baseUrl, model)`
       g. Return { chatFn, backend: "vllm", cleanup, costTracker }
    3. If neither vllmUrl nor runpodApiKey: throw error "Either --vllm-url or RUNPOD_API_KEY is required when backend is 'vllm'"

    **CLI changes (cli.ts):**

    Add dotenv import at top: `import 'dotenv/config';` (auto-loads .env file)

    Update the vllm-url validation logic:
    - Remove the hard requirement for --vllm-url when backend is vllm
    - Instead: if backend is vllm and no --vllm-url, check for RUNPOD_API_KEY env var
    - If neither exists, error: "Either --vllm-url or RUNPOD_API_KEY environment variable is required when --backend is vllm"

    Pass runpodApiKey to createBackend:
    ```typescript
    backendConfig = await createBackend(backend, {
      vllmUrl: opts.vllmUrl,
      vllmModel: opts.vllmModel,
      runpodApiKey: process.env.RUNPOD_API_KEY,
    });
    ```

    Add signal handler teardown (after backendConfig is created, before pipeline runs):
    ```typescript
    let cleanedUp = false;
    const doCleanup = async () => {
      if (cleanedUp) return;
      cleanedUp = true;
      if (backendConfig.cleanup) {
        console.log("\nTearing down cloud resources...");
        await backendConfig.cleanup();
        console.log("Cloud resources torn down.");
      }
    };

    process.on("SIGINT", async () => { await doCleanup(); process.exit(130); });
    process.on("SIGTERM", async () => { await doCleanup(); process.exit(143); });
    ```

    Wrap pipeline execution in try/finally:
    ```typescript
    let pipelineResult: PipelineResult;
    try {
      // Start cost tracking if cloud backend
      if (backendConfig.costTracker) {
        backendConfig.costTracker.start();
      }

      pipelineResult = await runPipeline(opts.input, { ... }, logger);
    } finally {
      // Stop cost tracking
      if (backendConfig.costTracker) {
        backendConfig.costTracker.stop();
      }
      // Always tear down cloud resources
      await doCleanup();
    }
    ```

    Add cost summary display after pipeline completion (before the final "Results written to" line):
    ```typescript
    if (backendConfig.costTracker) {
      const cost = backendConfig.costTracker.summary();
      console.log();
      console.log("=== Cloud Cost ===");
      console.log(`GPU time:    ${cost.gpuHours}`);
      console.log(`Est. cost:   ${cost.estimatedCost}`);
      console.log(`Rate:        $${cost.ratePerHour}/hr`);
    }
    ```

    Display cloud provisioning info in Pipeline section:
    ```typescript
    if (backend === "vllm" && !opts.vllmUrl) {
      console.log(`Backend:     vllm (RunPod cloud)`);
      console.log(`Endpoint:    ${/* endpointId from backendConfig */}`);
    } else if (backend === "vllm") {
      console.log(`Backend:     vllm (user-managed)`);
      console.log(`vLLM URL:    ${opts.vllmUrl}`);
    }
    ```

    Add .env to .gitignore if not already present.

    - Add test: vllm backend without vllmUrl and with runpodApiKey provisions via cloud provider (mock provider)
    - Add test: vllm backend without vllmUrl and without runpodApiKey throws descriptive error
    - Add test: cleanup function calls provider.teardown
    - Mock the cloud-provider module to avoid real API calls
