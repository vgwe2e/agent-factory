Create `src/scoring/schema-translator.ts` exporting:

1. `translateToResponseFormat(ollamaFormat: Record<string, unknown>): VllmResponseFormat` -- Takes the raw JSON schema object that Ollama receives as `format` and wraps it in vLLM's OpenAI-compatible response_format structure. Must strip `$schema` key from the schema if present (zodToJsonSchema includes it but vLLM's json_schema mode rejects it). Also strip `additionalProperties` if set to `false` at root level (vLLM xgrammar may choke on it -- research flag from STATE.md).

2. `validateScoringSchemas(): SchemaValidationResult` -- Imports the three JSON schemas from `schemas.ts` (technicalJsonSchema, adoptionJsonSchema, valueJsonSchema), translates each with `translateToResponseFormat`, and performs structural checks:

   - Must not use unsupported JSON schema keywords: `patternProperties`, `if`/`then`/`else`, `$ref`, `oneOf`, `anyOf` (vLLM xgrammar limitations)
   - Returns `{ valid: true }` or `{ valid: false, errors: Array<{ schema: string, reason: string }> }`

Type the VllmResponseFormat as:
