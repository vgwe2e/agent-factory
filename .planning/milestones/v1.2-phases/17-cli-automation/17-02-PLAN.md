    **4. Restructure the try/finally block (lines 205-233) into a try/catch/finally with retry loop:**

    ```
    try {
      costTracker?.start();

      // Initial pipeline run
      let lastResult = await runPipeline(opts.input, pipelineOptions, logger);

      // Retry loop (AUTO-01)
      const maxRetries = parseInt(opts.retry ?? "0", 10);
      for (let attempt = 1; attempt <= maxRetries; attempt++) {
        if (lastResult.errorCount === 0) break;

        const cleared = clearCheckpointErrors(opts.outputDir);
        console.log(`\n=== Retry ${attempt}/${maxRetries}: ${cleared} errored opportunities ===\n`);

        lastResult = await runPipeline(opts.input, {
          ...pipelineOptions,
          concurrency: 1,  // Always concurrency 1 for retries (locked decision)
        }, logger);
      }

      pipelineResult = lastResult;
    } catch (err) {
      // Fatal failure — ALWAYS tear down to prevent orphaned pods (AUTO-02)
      await doCleanup();
      const msg = err instanceof Error ? err.message : String(err);
      console.error(`${RED}Fatal: ${msg}${RESET}`);
      process.exit(2);  // EXIT CODE 2 = fatal (AUTO-04)
    } finally {
      costTracker?.stop();
      // Teardown only if --teardown flag set (AUTO-02)
      if (opts.teardown) {
        await doCleanup();
      }
    }
    ```

    **5. Import clearCheckpointErrors** at the top of cli.ts:
    `import { clearCheckpointErrors } from "./infra/checkpoint.js";`

    **6. Change teardown behavior (AUTO-02):**
    - Remove the unconditional `await doCleanup()` from the finally block (current line 232)
    - In the finally block: only call `doCleanup()` if `opts.teardown` is set
    - In the catch block (fatal): ALWAYS call `doCleanup()` regardless of flag (don't leave orphaned pods)
    - Signal handlers (SIGINT/SIGTERM, line 178-179) keep their existing behavior (always clean up on signal)
