    **1. PROV-01 -- GraphQL mutation fix (lines 121-138):**
    Replace the `env: [{ key: "MODEL_NAME", value: "${model}" }]` block with:
    ```
    dockerArgs: "--model ${model} --max-model-len 16384 --dtype auto"
    ```
    The `dockerArgs` field is a single string passed as Docker CMD args to the vLLM container. Use 16384 context length (fits Qwen 32B on H100 80GB comfortably) and `auto` dtype (lets vLLM pick bfloat16 on H100).

    **2. PROV-02 -- Model validation in healthCheck (lines 217-228):**
    Change healthCheck to parse the response body:
    - Parse JSON as `{ data: Array<{ id: string }> }` (OpenAI /v1/models format)
    - Extract model IDs from `data` array
    - Case-insensitive check: `modelIds.some(id => id.toLowerCase().includes(model.toLowerCase()))`
    - Return true only if match found
    - On no match, return false (provision() loop will eventually timeout)

    Also update provision()'s health poll loop (lines 191-214) to do a final model validation after health loop times out:
    - Before throwing the timeout error, do one last /v1/models fetch
    - If models are returned but none match, throw a specific model mismatch error: "Expected model '{model}' but loaded: [{actual model IDs}]. Use --vllm-url <url> to connect to a manually created pod."
    - If no models returned or fetch fails, throw the existing timeout error but with enriched message

    **3. PROV-03 -- Timeout and error improvements:**
    - Change DEFAULT_PROVISION_TIMEOUT_MS from 300_000 (5 min) to 600_000 (10 min)
    - Change DEFAULT_HEALTH_TIMEOUT_MS from 180_000 (3 min) to 300_000 (5 min)
    - Combined = 15 min total per requirement
    - All timeout/failure error messages must include:
      - The endpoint ID (already partially done)
      - Suggested recovery: "Check RunPod console for endpoint {id}. Verify GPU availability. Use --vllm-url <url> to connect to a manually created pod."
    - Wrap the entire provision() body (after apiKey check) in try/catch. In the catch block, attempt teardown of the endpoint (if endpointId was assigned) before re-throwing. This prevents orphaned endpoints.
