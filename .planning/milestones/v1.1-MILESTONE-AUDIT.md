## E2E Flow Verification

### Flow 1: Full Cloud (--backend vllm + RUNPOD_API_KEY)
`cli.ts` → `createBackend("vllm", {runpodApiKey})` → `validateScoringSchemas()` → `createCloudProvider().provision()` → health poll → `createVllmChatFn(baseUrl)` → `runPipeline({chatFn, concurrency, costTracker})` → `Semaphore(N)` + `Promise.allSettled` → scoring → `costTracker.summary()` → `cloud-cost.json` → `doCleanup()` → `provider.teardown()` → terminal cost display
**Status:** COMPLETE

### Flow 2: User-Managed vLLM (--backend vllm --vllm-url URL --concurrency N)
`cli.ts` → `createBackend("vllm", {vllmUrl})` → `validateScoringSchemas()` → `createVllmChatFn(vllmUrl)` → `runPipeline({chatFn, concurrency})` → concurrent scoring → no teardown, no cost tracking
**Status:** COMPLETE

### Flow 3: Default Ollama (no --backend flag)
`cli.ts` → `createBackend("ollama")` → `ollamaChat` → `runPipeline({chatFn, backend: "ollama"})` → `useLocalModels=true` → ModelManager → sequential scoring (concurrency=1)
**Status:** COMPLETE

### Flow 4: Crash Recovery
Mid-run crash → checkpoint.json preserved via atomic rename → resume → `loadCheckpoint()` → `getCompletedNames()` → skip scored items
**Status:** COMPLETE
