# Requirements: Aera Skill Feasibility Engine v1.1

**Defined:** 2026-03-11
**Core Value:** Produce actionable, adoption-realistic implementation specs for Aera skills -- not just technically feasible ones, but ones real users will actually adopt.

## v1.1 Requirements

Requirements for cloud-accelerated scoring milestone. Each maps to roadmap phases.

### vLLM Client

- [x] **VLLM-01**: User can score opportunities via vLLM backend using same ChatFn interface
- [x] **VLLM-02**: Engine translates Zod JSON schemas to vLLM response_format with json_schema
- [x] **VLLM-03**: User can select backend via `--backend ollama|vllm` CLI flag (default: ollama)
- [x] **VLLM-04**: Engine validates scoring schemas against vLLM before starting pipeline

### Concurrent Pipeline

- [x] **CONC-01**: Engine scores multiple opportunities simultaneously with configurable concurrency
- [x] **CONC-02**: Checkpoint system handles concurrent writes without corruption
- [x] **CONC-03**: Engine enforces per-request timeout to prevent semaphore starvation
- [x] **CONC-04**: User sees aggregated progress during concurrent scoring (in-flight, completed, errors)

### Cloud Infrastructure

- [x] **CLOUD-01**: Engine programmatically provisions ephemeral H100 endpoint via RunPod API
- [x] **CLOUD-02**: Engine auto-tears-down cloud instance on completion, crash, or signal (SIGINT/SIGTERM)
- [x] **CLOUD-03**: Engine tracks and reports GPU-hours consumed and estimated cost per run

## Future Requirements

Deferred to future milestone. Tracked but not in current roadmap.

### Validation

- **VAL-01**: Full Ford 339-opportunity E2E run on cloud backend with output verification
- **VAL-02**: Golden test suite comparing Ollama vs vLLM scores on 10 opportunities (tolerance band)
- **VAL-03**: Documented performance benchmarks (throughput, latency, cost per run)

### Advanced

- **ADVN-01**: Adaptive concurrency (dynamically resize semaphore based on throughput)
- **ADVN-02**: Multi-GPU / multi-instance support for larger evaluations
- **ADVN-03**: Generate full implementation specs for simulated opportunities
- **ADVN-04**: Model recommendations for local model to skill matching
- **ADVN-05**: Configurable scoring weights via CLI flags
- **ADVN-06**: Partial re-evaluation of specific opportunities

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Replace local Ollama path | Core constraint -- offline mode must always work |
| RunPod web UI workflow | Automating programmatically instead |
| openai npm SDK | Raw fetch is simpler, matches existing Ollama client pattern |
| Multi-GPU / multi-instance | Single H100 sufficient for Qwen 30B |
| Real-time streaming | Batch pattern validated in v1.0 |
