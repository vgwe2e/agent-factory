    - Writes output files: decision-flow.mmd, component-map.yaml, mock-test.yaml, integration-surface.yaml
    - Uses js-yaml dump() for YAML file serialization (not string concatenation)
    - Creates evaluation/simulations/<slug>/ directory for each opportunity
    - Returns SimulationPipelineResult with: results array, totalSimulated count, totalFailed count, aggregated confirmed/inferred counts
    - Pipeline builds knowledge index once (via buildKnowledgeIndex) and reuses across all opportunities
    - Empty inputs array returns immediately with zero counts
