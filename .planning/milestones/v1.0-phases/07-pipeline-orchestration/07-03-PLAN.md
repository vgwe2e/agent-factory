    - Pipeline scores in tier-priority order (Tier 1 first, skip action="skip")
    - Pipeline catches per-opportunity scoring errors and records them, continues to next
    - Pipeline calls archiveAndReset every N opportunities (configurable, default 25)
    - Pipeline calls unloadAll at the end
    - Pipeline logs start, stage transitions, per-opportunity progress, completion summary
    - PipelineResult includes triageCount, scoredCount, promotedCount, skippedCount, errorCount, totalDurationMs, errors array
  </behavior>
  <action>
    1. Create src/pipeline/pipeline-runner.test.ts:
       - Use dependency injection to mock: chatFn (returns canned scoring JSON), fetchFn (for ModelManager), parseExport result (provide a minimal HierarchyExport fixture with 3-4 L3s in different tiers).
       - Test: pipeline processes all non-skipped opportunities and returns correct counts.
       - Test: pipeline records error and continues when chatFn returns { success: false }.
       - Test: pipeline calls archiveAndReset after threshold (set threshold to 2 for test).
       - Test: pipeline calls unloadAll at end.
       - Use createLogger("silent") for test logger.

    2. Create src/pipeline/pipeline-runner.ts:
       - Export PipelineOptions: { outputDir: string; logLevel?: string; archiveThreshold?: number; models?: { triage: string; scoring: string }; chatFn?: ChatFn; fetchFn?: typeof fetch }.
       - Export PipelineResult: { triageCount, scoredCount, promotedCount, skippedCount, errorCount, totalDurationMs, errors }.
       - Export async function runPipeline(inputPath: string, options: PipelineOptions, logger: Logger): Promise<PipelineResult>.
       - Implementation flow:
         a. Parse export (parseExport)
         b. Triage (triageOpportunities) -- pure function, no model needed
         c. Log triage summary (tier counts, skip counts)
         d. Create ModelManager and switch to scoring model
         e. Create EvaluationContext
         f. Load knowledge context (getComponentSummary, getPBNodeSummary)
         g. Group L4s by L3 (reuse groupL4sByL3)
         h. Loop processable triage results sorted by tier:
            - Create child logger with oppId and tier
            - Call scoreOneOpportunity with chatFn
            - If error result: addError to context, log, continue
            - If success: addResult to context, log composite and promoted status
            - Every archiveThreshold results: archiveAndReset
         i. Final archiveAndReset (flush remaining)
         j. unloadAll
         k. Log summary and return PipelineResult

    3. Wire chatFn through to scoreOneOpportunity. The chatFn in PipelineOptions creates a curried version that binds the scoring model name via the refactored ollamaChat model parameter.

    4. Run tests.
  </action>
